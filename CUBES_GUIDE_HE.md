## סקירת המערכת

**12-flow** היא מערכת ויזואלית לבניית תהליכי ניתוח טיסות (Workflows), חלק ממוצר Tracer 42.

המערכת מאפשרת למשתמשים לבנות צינורות עיבוד נתונים בצורה גרפית — **ללא צורך בכתיבת קוד** — על ידי גרירת קוביות עיבוד לקנבס, הגדרת פרמטרים, חיבור פלטים לקלטים, והרצה מול מסד הנתונים של Tracer 42.

## מה זה קוביה (Cube)?

**קוביה** היא יחידת עיבוד עצמאית שמבצעת פעולה אחת מוגדרת. כל קוביה מורכבת מ:

```
          ┌─────────────────────────────────┐
          │         שם הקוביה               │
          │        (קטגוריה)                │
          │                                 │
  קלט A ──┤►                           ◄├── פלט X
  קלט B ──┤►                           ◄├── פלט Y
  קלט C ──┤►       לוגיקת עיבוד        ◄├── פלט Z
          │                                 │
          │                            ◄├── Full Result
          └─────────────────────────────────┘
```

### מרכיבי קוביה

| מרכיב | תיאור |
|--------|--------|
| **מזהה** (`cube_id`) | שם טכני ייחודי, באנגלית, עם קו תחתון (למשל `all_flights`) |
| **שם תצוגה** (`name`) | השם שמוצג למשתמש בממשק (למשל "All Flights") |
| **תיאור** (`description`) | הסבר קצר על מה הקוביה עושה |
| **קטגוריה** (`category`) | סיווג הקוביה — משפיע על הצבע וההצגה בקטלוג |
| **פרמטרי קלט** (`inputs`) | הנתונים שהקוביה מקבלת כדי לעבוד |
| **פרמטרי פלט** (`outputs`) | הנתונים שהקוביה מחזירה לאחר העיבוד |
| **Widget** (`widget`) | רכיב ויזואלי מיוחד ברמת הקוביה (אופציונלי) |

### פלט מלא (Full Result)

לכל קוביה מתווסף אוטומטית פלט מיוחד בשם **Full Result** — הוא אוגד את כל הפלטים יחד לאובייקט JSON אחד. זה שימושי כדי להעביר את כל התוצאה לקוביה הבאה בפעולה אחת.

## קטגוריות קוביות

כל קוביה שייכת לאחת מחמש קטגוריות. הקטגוריה קובעת את הצבע בממשק ואת המיקום בקטלוג:

```
┌────────────────┬────────────┬──────────────────────────────────────────┐
│   קטגוריה      │    צבע     │   תיאור                                 │
├────────────────┼────────────┼──────────────────────────────────────────┤
│ 📊 DATA_SOURCE │   סגול     │ שליפת נתונים ממסד הנתונים               │
│ 🔍 FILTER      │   כתום     │ סינון וצמצום נתונים                      │
│ 🧮 ANALYSIS    │   סגול-כהה │ עיבוד, חישוב וניתוח                     │
│ 📈 AGGREGATION │   ציאן     │ צבירה, קיבוץ וסיכום נתונים              │
│ 📤 OUTPUT      │   ירוק     │ הצגת או ייצוא תוצאות                    │
└────────────────┴────────────┴──────────────────────────────────────────┘
```

### סדר הצגה בקטלוג

הקוביות מוצגות בסרגל הצד בסדר קבוע לפי קטגוריה:

```
  ┌─────────────────────┐
  │  🔍 חיפוש...        │
  ├─────────────────────┤
  │  📊 מקורות נתונים   │
  │  ├── All Flights    │

  │  ├── Get Flight...  │
  │  └── Get Learned... │

  │  🔍 סינון           │
  │  └── (ריק כרגע)     │
  ├─────────────────────┤
  │  🧮 ניתוח           │
  │  └── Add Numbers    │
  ├─────────────────────┤
  │  📈 צבירה           │
  │  └── Count By Field │
  ├─────────────────────┤
  │  📤 פלט             │

  └─────────────────────┘
```

## סוגי פרמטרים

כל פרמטר (קלט או פלט) מוגדר עם **סוג** שקובע מה הנתון ואיך הוא מוצג בממשק:

```
┌──────────────────┬──────────┬─────────────────────────────────────┐
│     סוג          │   צבע    │   דוגמה                             │
├──────────────────┼──────────┼─────────────────────────────────────┤
│ STRING           │   כחול   │ "LLBG", "EL-AL*"                   │
│ NUMBER           │   ירוק   │ 42, 3.14, 604800                   │
│ BOOLEAN          │   כתום   │ true / false                       │
│ LIST_OF_STRINGS  │   טורקיז │ ["ABC123", "DEF456"]               │
│ LIST_OF_NUMBERS  │   טורקיז │ [1, 2, 3]                          │
│ JSON_OBJECT      │   אפור   │ {"key": "value"} או מערך אובייקטים │
└──────────────────┴──────────┴─────────────────────────────────────┘
```

### Widget Hints — רכיבי קלט מיוחדים

חלק מהפרמטרים מקבלים **רמז ויזואלי** (`widget_hint`) שמשנה את הצורה שבה המשתמש מזין ערך:

| Widget Hint | רכיב בממשק | תיאור |
|-------------|------------|--------|
| `relative_time` | בוחר זמן יחסי | מספר + יחידה (שניות/דקות/שעות/ימים) |
| `datetime` | בוחר תאריך ושעה | שדה datetime-local, נשמר כ-epoch |
| `polygon` | ציור פוליגון על מפה | כפתור שפותח מפה — המשתמש מסמן נקודות |
| *(ללא)* | שדה רגיל | טקסט, מספר, צ'קבוקס לפי סוג הפרמטר |

## חיבורים בין קוביות

### עקרון החיבור

חיבורים נעשים **ברמת הפרמטר** — לא ברמת הקוביה. המשתמש מחבר פלט ספציפי של קוביה אחת לקלט ספציפי של קוביה אחרת.

```
  ┌──────────────┐          ┌──────────────┐
  │ All Flights  │          │ Get Anomalies│
  │              │          │              │
  │    flights ──┤          ├── flight_ids │
  │  flight_ids ─┼──────────┼►  flight_ids │
  │ Full Result ─┤          ├── min_sev.   │
  └──────────────┘          └──────────────┘
```

### כללי חיבור

- **סוגים שונים מותרים** — המערכת מציגה אזהרה אבל מאפשרת חיבור
- **ערך חיבור עוקף ערך ידני** — אם פרמטר מחובר, הערך שהוזן ידנית מתעלם ממנו
- **Full Result** — ניתן לחבר את הפלט המלא רק לקלט שמוגדר עם `accepts_full_result: true`
- **פרמטר מחובר** — מציג תגית "Connected" במקום שדה העריכה

## תהליך ההרצה

כשהמשתמש לוחץ "הרצה" (▶), המערכת מבצעת את התהליך הבא:

```
  1️⃣ מיון טופולוגי     2️⃣ הרצה בסדר        3️⃣ תוצאות
  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
  │ קוביה A      │    │ ⏳ ממתין     │    │ ✅ הושלם     │
  │   ↓          │    │ 🔄 רץ       │    │    52ms      │
  │ קוביה B      │──► │ ✅ הושלם    │──► │ ✅ הושלם     │
  │   ↓          │    │ 🔄 רץ       │    │    128ms     │
  │ קוביה C      │    │ ⏳ ממתין     │    │ ✅ הושלם     │
  └──────────────┘    └──────────────┘    └──────────────┘
```

1. **מיון טופולוגי** — המערכת מחשבת את סדר ההרצה לפי החיבורים (מזהה מעגלים)
2. **הרצה סדרתית** — כל קוביה רצה לפי הסדר, עם עדכוני סטטוס בזמן אמת (SSE)
3. **הגבלת שורות** — תוצאות מוגבלות ל-10,000 שורות לכל קוביה
4. **טיפול בשגיאות** — קוביה שנכשלת גורמת לדילוג על כל הקוביות שתלויות בה

### סטטוסי הרצה

| סטטוס | סמל | משמעות |
|--------|------|---------|
| `pending` | ⏳ | ממתין לתורו |
| `running` | 🔄 | רץ כרגע |
| `done` | ✅ | הסתיים בהצלחה |
| `error` | ❌ | נכשל (הודעת שגיאה מוצגת) |
| `skipped` | ➖ | דולג כי קוביה קודמת נכשלה |

**פרמטרי קלט:**

| `min_member_count` | NUMBER | לא | — | — | מספר טיסות חברות מינימלי (סינון נתיבים לא אמינים) |
| `output_geometry` | STRING | לא | `"centerline"` | — | `"centerline"` או `"corridor"` |
| `width_override` | NUMBER | לא | — | — | דריסת רוחב מסדרון במיילים ימיים |

**פרמטרי פלט:**

| שם | סוג | תיאור |

### 📈 Count By Field — ספירה לפי שדה

> קיבוץ מערך נתונים לפי שדה וספירת מופעים

**קטגוריה:** `AGGREGATION` (צבירה)

**מה הקוביה עושה:**
מקבלת מערך נתונים כלשהו (או Full Result), מקבצת לפי שדה שנבחר, וסופרת כמה פעמים כל ערך מופיע. התוצאות ממוינות מהגבוה לנמוך.

**דוגמה:** קבלת רשימת טיסות → קיבוץ לפי `airline` → תוצאה: `[{value: "EL AL", count: 15}, {value: "Lufthansa", count: 8}, ...]`

**פרמטרי קלט:**

| שם | סוג | חובה | Full Result | תיאור |
|----|------|-------|------------|--------|
| `data` | JSON_OBJECT | **כן** | **כן** ✅ | מערך אובייקטים — מקבל גם Full Result |
| `group_by_field` | STRING | **כן** | לא | שם השדה לקיבוץ |

**פרמטרי פלט:**

| שם | סוג | תיאור |
|----|------|--------|
| `counts` | JSON_OBJECT | מערך `{value, count}` ממוין לפי ספירה יורדת |
| `Full Result` | JSON_OBJECT | כל הפלטים באובייקט אחד (אוטומטי) |

### 🧮 Add Numbers — חיבור מספרים

**מה הקוביה עושה:**
קוביית עזר פשוטה שמחברת שני מספרים ומחזירה את הסכום.

**פרמטרי קלט:**

|----|------|-------|--------|
| `a` | NUMBER | לא | מספר ראשון |
| `b` | NUMBER | לא | מספר שני |

**פרמטרי פלט:**

| שם | סוג | תיאור |
|----|------|--------|
| `sum` | NUMBER | סכום של a ו-b |
| `Full Result` | JSON_OBJECT | כל הפלטים באובייקט אחד (אוטומטי) |

### תכונות עיקריות

| **חיבור ויזואלי** | חיבור פלטים לקלטים באמצעות גרירת קווים בין הידיות (handles) |

| **זמן יחסי** | בחירת טווח זמן נוח (למשל "7 ימים אחרונים") |
| **בחירת תאריך** | בוחר תאריך ושעה לסינון מדויק |
| **תגיות** | הזנת רשימות (כמו flight_ids) ככיפסים נוחים |
