## Color

All values sourced directly from `src/styles/theme.css`. No new color values are introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--color-bg)` = `#000000` | App background, canvas area |
| Secondary (30%) | `var(--color-surface)` = `#08090d` / `var(--color-surface-raised)` = `#0e1017` | Chat panel background, message bubbles, header bar |
| Accent (10%) | `var(--color-accent)` = `#6366f1` | See reserved list below |

Accent (`#6366f1`) reserved for:
- Active mode segment in the three-way mode toggle (Optimize / Fix / General)

- Text selection highlight (inherited from `::selection` in index.css)
- Focus ring on chat textarea: `0 0 0 3px var(--color-accent-glow)` (matches sidebar search focus)

No new accent colors. Error-fix mode uses `var(--color-error)` for its mode indicator only — the mode badge text color, not the segment background.

## Layout Contract

Pre-populated from D-01 through D-04 in 20-CONTEXT.md.

### Editor Layout with Chat Panel Open

```
[ Toolbar — 52px height, full width ]
[ CubeCatalog 280px ] [ Canvas — flex:1 ] [ ChatPanel 320px default ]
                       [ IssuesPanel ]
```

- Chat panel sits inside `.app__body` as a flex sibling to `.app__canvas-area`, placed to the right.
- Canvas area (`flex:1`) shrinks between both sidebars — canvas does not scroll, it compresses.
- Chat panel: `height: 100%`, `flex-shrink: 0`, `background: linear-gradient(180deg, var(--color-surface) 0%, #050609 100%)` — mirrors CubeCatalog exactly but mirrored on the right edge.
- Panel left edge gradient line: same animated gradient as `.sidebar::after` but positioned on `left` edge instead of `right`.

### Chat Panel Internal Structure

```
[ Panel Header: "AGENT" title + mode toggle (Optimize|Fix|General) + close button ]
[ Message List — flex:1, overflow-y:auto, padding: 12px 0 ]
[ Diff Proposal Block — shown when agent proposes changes ]
[ Chat Input Row — textarea + Send button, flex-shrink: 0 ]
```

### Panel Header

- Height: 44px (8px taller than IssuesPanel header to accommodate mode toggle)
- Background: `var(--color-surface-raised)`
- Border bottom: `1px solid var(--color-border)`
- Title "AGENT": 11px, weight 600, uppercase, letter-spacing 0.06em, color `var(--color-text-muted)` — matches `.sidebar__title` but right-aligned panel so title on left
- Mode toggle: three segments `Optimize | Fix | General`, each 28px tall, font-size 11px, weight 600
- Close button: 28x28px, same style as `.sidebar__toggle`

### Mode Toggle Segments

- Container: `display: flex`, `border-radius: var(--radius-sm)`, `background: rgba(255,255,255,0.04)`, `border: 1px solid var(--color-border)`, `padding: 2px`
- Each segment: `padding: 0 10px`, `height: 24px`, `border-radius: 6px`, `font-size: 11px`, `font-weight: 600`, `cursor: pointer`
- Inactive: `color: var(--color-text-dim)`, transparent background
- Active: `color: var(--color-text)`, `background: rgba(var(--accent-rgb), 0.18)`, `border: 1px solid rgba(var(--accent-rgb), 0.25)` — matches `btn-accent-bg/border`
- Error-Fix active: active segment additionally shows a 4px colored dot using `var(--color-error)` at left of "Fix" label when error count > 0

### Drag Handle

- 8px wide strip on the left edge of the chat panel
- Visual: `background: transparent` normally, `background: rgba(255,255,255,0.04)` on hover
- Cursor: `col-resize`
- No border — the panel's animated left-edge gradient line serves as visual separator

### Toolbar Chat Toggle Button

- Placed in `.toolbar__actions` between the settings gear button and Save button
- Style: matches `.toolbar__help-btn` exactly — 28x28px circle, `border-radius: 50%`
- Icon: chat bubble SVG, 14x14, inline, stroke-based, `stroke-width: 1.5`
- Active state (panel open): shows a 6px accent dot at top-right of button — `position: absolute`, `background: var(--color-accent)`, `border-radius: 50%`, matches `.toolbar__dirty-dot` approach
- Badge state (agent has suggestions pending): uses same 6px dot but pulsing — `animation: cancel-pulse 2s ease-in-out infinite` adapted with accent color

## Auto-Open Fix Mode Behavior

When a workflow execution finishes with errors (detected via existing `executionErrors` in flowStore):

1. Panel opens: `chatPanelOpen = true`
2. Mode switches to Fix: `chatPanelMode = 'fix'`
3. An initial message is injected into the message thread — pre-written template, not a Gemini call:
   - Text: `I see errors in {N} cube(s) from the last run. Want me to diagnose the issues and suggest a fix?`
   - Displayed as an agent message bubble (left-aligned)
   - Below this message: a single "Diagnose" button styled as `.glass-btn.glass-btn--accent` with label "Diagnose Errors" — clicking it sends an automatic first message to the SSE endpoint

This pre-written template approach (not a Gemini call) is the implementation strategy per Claude's Discretion in CONTEXT.md.

## Panel Collapsed / Icon-Strip State

When the chat panel is closed:
- It does not render an icon strip (unlike CubeCatalog)
- It fully hides (`display: none` or `width: 0` with `overflow: hidden`)
- The toolbar button alone serves as the re-open mechanism
- Rationale: chat icon strip provides no value at 48px — no category icons to show

## Interaction States Summary

| State | Visual |
|-------|--------|
| Panel closed | Not rendered, toolbar button shows no dot |
| Panel open, no messages | Empty state shown in message list |
| Panel open, agent streaming | Streaming cursor on agent bubble, Send button disabled |
| Panel open, diff proposed | DiffProposal block appears below last agent message |
| Panel open, diff applied | DiffProposal block shows "Applied" label, Apply/Reject buttons hidden |
| Panel open, diff rejected | DiffProposal block removed from thread |
| Fix mode, errors present | Error dot on "Fix" segment, auto-open behavior fires |
| Fix mode, no errors | Fix mode accessible manually, no auto-open behavior |
