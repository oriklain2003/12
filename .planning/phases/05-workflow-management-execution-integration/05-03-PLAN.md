From backend SSE endpoint:
- URL: GET /api/workflows/{id}/run/stream (via Vite proxy: /api/workflows/{id}/run/stream)
- Event type: "cube_status"
- Data: JSON-serialized CubeStatusEvent
- Stream sends "pending" for all nodes first, then "running" -> "done"/"error"/"skipped" transitions
- Server closes stream when all nodes reach terminal state
- CRITICAL: EventSource auto-reconnects on server close — must explicitly close on client when all terminal

1. Create `frontend/src/hooks/useWorkflowSSE.ts`:
   - Export `useWorkflowSSE()` hook returning `{ startStream, stopStream }`.
   - Use `useRef<EventSource | null>(null)` to hold the connection.
   - `startStream(workflowId: string)`:
     - Close existing connection if any.
     - Call `useFlowStore.getState().startExecution()`.
     - Create `new EventSource('/api/workflows/' + workflowId + '/run/stream')`.
     - Listen for `'cube_status'` events (NOT the default `'message'` event — use `es.addEventListener('cube_status', handler)`).
     - In handler: `JSON.parse(event.data)` as `CubeStatusEvent`, call `useFlowStore.getState().setNodeExecutionStatus(data.node_id, data)`.
     - After each event, check completion: read `completedCount` and `totalCount` from store. If `completedCount >= totalCount && totalCount > 0`, call `es.close()` and `useFlowStore.getState().stopExecution()`.
     - On `es.onerror`: close connection, call `stopExecution()`.
   - `stopStream()`: close EventSource ref, null it, call `stopExecution()`.
   - Use `useCallback` for both functions.
