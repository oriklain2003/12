  <action>
1. Create `frontend/src/pages/DashboardPage.tsx` as a single-file component (no separate WorkflowList/WorkflowCard files — keep it simple, the card is only used here):

   **State:**
   - `workflows: WorkflowResponse[]` — fetched from API
   - `loading: boolean` — true during initial fetch
   - `renamingId: string | null` — which card is in rename mode
   - `renameValue: string` — current rename input value
   - `deletingId: string | null` — which card shows delete confirmation

   **On mount (useEffect):**
   - Call `getWorkflows()`, set workflows, set loading=false.
   - On error, show `toast.error('Failed to load workflows')`.

   **Card grid layout:**
   - Use CSS Grid: `grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))` with gap.
   - Each card has `glass` class for the dark glass effect.
   - Card content:
     - Workflow name (or rename input if renamingId matches)
     - Created date (formatted: `new Date(wf.created_at).toLocaleDateString()`)
     - Updated date (formatted similarly)
     - Actions row at bottom: Rename button, Delete button
   - Click card body (not action buttons) navigates to `/workflow/${wf.id}` using `useNavigate`.

   **Rename flow:**
   - Click Rename button -> sets renamingId and renameValue.
   - Input field replaces the name text, auto-focused.
   - Enter or blur -> calls `updateWorkflow(id, renameValue, wf.graph_json)` to save. On success, update local workflows state. Clear renamingId.
   - Escape -> cancel rename (clear renamingId).

   **Delete flow (per user decision: simple, easy, no complex modal):**
   - Click Delete button -> sets `deletingId` to that card's id, shows inline "Delete?" confirmation text with Confirm/Cancel buttons replacing the normal action row.
   - Confirm -> calls `deleteWorkflow(id)`, removes from local state, shows `toast.success('Workflow deleted')`.
   - Cancel -> clears deletingId.
   - Keep it lightweight — no modal, no overlay.

   **Empty state:**
   - If `workflows.length === 0` and not loading: show centered "No saved workflows" text with a "Create New Workflow" button below.

   **Header:**
   - Page header area with "Workflows" title and a "New Workflow" button (navigates to `/workflow/new`).
   - Style with dark theme variables. The page background should use `--color-bg`.

   **Page layout:**
   - Full-page layout (no sidebar, no toolbar — dashboard is standalone).
   - Centered content area with max-width ~1200px and horizontal padding.
   - Import `Toaster` from sonner and render it (for delete/rename error toasts).
