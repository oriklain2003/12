  <action>
1. Create `frontend/src/types/execution.ts` mirroring the backend CubeStatusEvent schema:
   ```typescript
   export type CubeStatus = 'pending' | 'running' | 'done' | 'error' | 'skipped';
   export interface CubeStatusEvent {
     node_id: string;
     status: CubeStatus;
     outputs?: Record<string, unknown>;
     truncated?: boolean;
     error?: string;
     reason?: string;
   }
   ```

2. Extend `frontend/src/store/flowStore.ts` FlowState interface with new fields:
   - `workflowId: string | null` (null = unsaved new workflow)
   - `workflowName: string` (default 'Untitled Workflow')
   - `isDirty: boolean` (default false)
   - `isRunning: boolean` (default false)
   - `executionStatus: Record<string, { status: CubeStatus; error?: string; outputs?: Record<string, unknown> }>`
   - `completedCount: number` (default 0)
   - `totalCount: number` (default 0)

3. Add new actions to FlowState:
   - `setWorkflowName(name: string)` — sets name and marks dirty
   - `setWorkflowMeta(id: string | null, name: string)` — sets id + name without marking dirty (used by load)
   - `saveWorkflow(): Promise<string>` — serializes nodes/edges to WorkflowGraph using serialization helpers below, calls `createWorkflow` if workflowId===null or `updateWorkflow` if existing. Returns the workflow ID. Sets `isDirty = false` on success.
   - `loadWorkflow(id: string): Promise<void>` — calls `getWorkflow(id)`, waits for catalog to be loaded (check `catalog.length > 0`), deserializes graph_json nodes back into CubeFlowNode[] by re-hydrating `cubeDef` from catalog. Sets nodes, edges, workflowId, workflowName, isDirty=false, clears results and executionStatus.
   - `resetWorkflow()` — clears nodes, edges, results, executionStatus, sets workflowId=null, workflowName='Untitled Workflow', isDirty=false.
   - `startExecution()` — sets isRunning=true, clears executionStatus, completedCount=0, totalCount=0.
   - `stopExecution()` — sets isRunning=false.
   - `setNodeExecutionStatus(nodeId, event)` — updates executionStatus[nodeId] with the event data. If status is 'pending', increment totalCount. If status is 'done', 'error', or 'skipped', increment completedCount. If event has outputs, call setResults(nodeId, ...) with the outputs.

4. Modify existing `onNodesChange`, `onEdgesChange`, `updateNodeParam` to also set `isDirty = true` when changes occur.

5. Add serialization helpers (can be module-level functions in flowStore.ts or a separate utils file):
   - `serializeGraph(nodes: CubeFlowNode[], edges: Edge[]): WorkflowGraph` — maps nodes to WorkflowNode (id, type:'cube', position, data: {cube_id, params}), maps edges to WorkflowEdge (id, source, target, sourceHandle, targetHandle).
   - `deserializeGraph(graph: WorkflowGraph, catalog: CubeDefinition[]): { nodes: CubeFlowNode[], edges: Edge[] }` — maps WorkflowNodes back to CubeFlowNodes by looking up cubeDef from catalog via cube_id. For edges, map back to Edge[] preserving type assignment (set type to 'straight' default; mismatch detection can happen on reconnection).

6. Import `getWorkflow`, `createWorkflow`, `updateWorkflow` from `../api/workflows` in flowStore.ts.
7. Import `CubeStatus` from `../types/execution` in flowStore.ts.

IMPORTANT: The `loadWorkflow` action must guard against catalog not yet loaded. If `catalog.length === 0`, it should fetch the catalog first using `fetchCatalog()` from api/cubes.ts before proceeding with deserialization. Read the existing catalog-fetching pattern from `CubeCatalog.tsx` to understand how catalog is loaded (it calls `fetchCatalog` from `api/cubes.ts` and dispatches to `setCatalog`). In the store's `loadWorkflow`, if catalog is empty, call `fetchCatalog()` from the API and `setCatalog()` before deserializing.
