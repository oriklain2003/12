Create the central Zustand store at `frontend/src/store/flowStore.ts`. This is the single source of truth for all canvas state.

**Type definitions (define at top of file, NOT in a separate types file — keep collocated):**

```typescript
import type { Node, Edge, NodeChange, EdgeChange, Connection } from '@xyflow/react';
import type { CubeDefinition } from '../types/cube';

export type CubeNodeData = {
  cube_id: string;
  cubeDef: CubeDefinition;
  params: Record<string, unknown>;
};

export type CubeFlowNode = Node<CubeNodeData, 'cube'>;
```

**Store interface — must include all of these:**

State:
- `nodes: CubeFlowNode[]` — all nodes on canvas
- `edges: Edge[]` — all edges/connections
- `catalog: CubeDefinition[]` — loaded from backend
- `results: Record<string, { rows: unknown[]; truncated: boolean }>` — per-node results keyed by node ID
- `catalogLoading: boolean` — loading indicator

Actions:
- `setCatalog(catalog: CubeDefinition[]): void` — set catalog after fetch
- `setCatalogLoading(loading: boolean): void`
- `onNodesChange(changes: NodeChange<CubeFlowNode>[]): void` — delegates to `applyNodeChanges`
- `onEdgesChange(changes: EdgeChange[]): void` — delegates to `applyEdgeChanges`
- `onConnect(connection: Connection): void` — delegates to `addEdge` (basic version; validation happens in FlowCanvas)
- `addCubeNode(cubeId: string, position: { x: number; y: number }): void` — lookup cubeDef from catalog, create node with crypto.randomUUID() as id, type='cube', initialize params to defaults from cubeDef.inputs
- `updateNodeParam(nodeId: string, paramName: string, value: unknown): void` — immutable update of params nested in node data
- `setResults(nodeId: string, rows: unknown[], truncated: boolean): void` — store results for a node
- `clearResults(): void` — clear all results (before new execution)

**Implementation notes:**
- Use `create<FlowState>()((set, get) => ({ ... }))` — Zustand v5 double-parentheses pattern.
- Import `applyNodeChanges`, `applyEdgeChanges`, `addEdge` from `@xyflow/react`.
- In `addCubeNode`: initialize `params` by iterating cubeDef.inputs and setting default values from `param.default ?? (param.type === ParamType.BOOLEAN ? false : undefined)`.
- Use `get()` inside actions to read current state — never import the store itself (avoids circular dependency, per RESEARCH pitfall 6).
- All node/edge updates MUST create new object references (spread, not mutate).

**1. ParamHandle.tsx — color-coded handle component:**

```typescript
export const PARAM_COLORS: Record<ParamType, string> = {
  [ParamType.STRING]: 'var(--color-param-string)',       // blue
  [ParamType.NUMBER]: 'var(--color-param-number)',       // green
  [ParamType.BOOLEAN]: 'var(--color-param-boolean)',     // orange
  [ParamType.LIST_OF_STRINGS]: 'var(--color-param-list)', // teal
  [ParamType.LIST_OF_NUMBERS]: 'var(--color-param-list)', // teal
  [ParamType.JSON_OBJECT]: 'var(--color-param-json)',    // gray
};
```

Props: `param: ParamDefinition`, `type: 'source' | 'target'`, `isConnectable: boolean`.
Renders a React Flow `<Handle>` with:
- `type` and `position` (Left for target, Right for source)
- `id={param.name}` (CRITICAL — must match for connection validation)
- `style={{ background: PARAM_COLORS[param.type], width: 10, height: 10 }}`
- Apply `className="nodrag"` so dragging the handle doesn't drag the node

Also render a small label next to the handle showing `param.name` in dim text.

**2. ParamField.tsx — inline parameter editor:**

Props: `nodeId: string`, `param: ParamDefinition`.

Derive `isConnected` from the store using a Zustand selector:
```typescript
const isConnected = useFlowStore(
  (s) => s.edges.some((e) => e.target === nodeId && e.targetHandle === param.name)
);
```

When `isConnected` is true: render a muted "Connected" label with a small link icon instead of the input. Do NOT render the input at all.

When NOT connected, render based on `param.type`:
- `STRING`: `<input type="text">` with placeholder from param.description
- `NUMBER`: `<input type="number">`
- `BOOLEAN`: `<input type="checkbox">`
- `LIST_OF_STRINGS`: Use `react-tag-input-component` — `<TagsInput>` component. Value is stored as string[] in params.
- `LIST_OF_NUMBERS`: Use `react-tag-input-component` — `<TagsInput>`. On change, parse each tag to Number. Value stored as number[] in params.
- `JSON_OBJECT`: `<textarea rows={2}>` with monospace font. Value stored as string in params; compact, minimal height.

All editors call `useFlowStore.getState().updateNodeParam(nodeId, param.name, value)` on change.

Apply `className="nodrag nowheel"` on all inputs so they don't interfere with node dragging or canvas zoom.

Style with ParamField.css: compact height, font-size 11px, inputs take full width of param row.

**3. ResultsPanel.tsx — compact results preview:**

Read results from store:

const result = useFlowStore((s) => s.results[nodeId]);
