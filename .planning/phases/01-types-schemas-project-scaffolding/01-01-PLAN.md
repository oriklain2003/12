  <name>Task 1: Create cube type system schemas and BaseCube abstract class</name>
  <files>backend/app/schemas/cube.py, backend/app/cubes/base.py</files>
  <action>
Create `backend/app/schemas/cube.py` with Pydantic models:

1. `ParamType(str, Enum)` with values: `string`, `number`, `boolean`, `list_of_strings`, `list_of_numbers`, `json_object`
2. `CubeCategory(str, Enum)` with values: `data_source`, `filter`, `analysis`, `aggregation`, `output`
3. `ParamDefinition(BaseModel)` with fields:
   - `name: str` — unique param name
   - `type: ParamType` — the data type
   - `required: bool = False`
   - `default: Any = None`
   - `description: str = ""`
   - `accepts_full_result: bool = False` — (input only) whether this param can receive a Full Result connection
4. `CubeDefinition(BaseModel)` with fields:
   - `cube_id: str` — unique identifier
   - `name: str` — display name
   - `description: str = ""`
   - `category: CubeCategory`
   - `inputs: list[ParamDefinition] = []`
   - `outputs: list[ParamDefinition] = []`

Create `backend/app/cubes/base.py` with:

1. `BaseCube` as an abstract class (use `abc.ABC` and `abc.abstractmethod`):
   - Class-level properties: `cube_id: str`, `name: str`, `description: str`, `category: CubeCategory`, `inputs: list[ParamDefinition]`, `outputs: list[ParamDefinition]`
   - `definition() -> CubeDefinition` property method that constructs a CubeDefinition from the class properties. This method MUST automatically append a `__full_result__` ParamDefinition to outputs (type=json_object, name="__full_result__", description="Full result bundle of all outputs") so every cube always exposes the Full Result port.
   - `async execute(self, **inputs) -> dict[str, Any]` abstract method — subclasses implement their logic here, returning a dict of output param names to values.

Import from `app.schemas.cube` — do NOT re-define types.
  </action>
