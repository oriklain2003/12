    ```python
    class NoRecordedTakeoffCube(BaseCube):
        cube_id = "no_recorded_takeoff"
        name = "No Recorded Takeoff"
        description = (
            "Detects flights with no recorded takeoff by checking if the first track "
            "point in normal_tracks is at or above a configurable altitude threshold. "
            "Establishes the behavioral output schema for downstream detection cubes."
        )
        category = CubeCategory.ANALYSIS
    ```

    **Inputs** (per D-05, INFRA-03):
    - `flight_ids`: LIST_OF_STRINGS, required=False, description="Flight IDs to analyze"
    - `full_result`: JSON_OBJECT, required=False, accepts_full_result=True, description="Full result from upstream cube"
    - `altitude_threshold_ft`: NUMBER, required=False, default=300, description="Altitude threshold in feet. Flights whose first track point is at or above this altitude are flagged. Default: 300."
    - Plus `TIME_MODE_PARAMS` from time_utils (spread into inputs list: `*TIME_MODE_PARAMS`)

    **Outputs** (per D-09):
    - Single `results`: JSON_OBJECT, description="List of flight dicts with behavioral fields (no_recorded_takeoff, deviation_score, diagnostic)"

    **execute() logic:**

    1. **Extract inputs:**
       - `altitude_threshold_ft = float(inputs.get("altitude_threshold_ft") or 300)`
       - Extract `flight_ids` list: try direct `inputs.get("flight_ids")` first, then fall back to `inputs.get("full_result", {}).get("flight_ids", [])`
       - Extract upstream `results` rows from full_result for passthrough: `upstream_rows = inputs.get("full_result", {}).get("results", [])`
       - Build `upstream_by_id = {row["flight_id"]: row for row in upstream_rows}` lookup

    2. **Empty input guard:** If no flight_ids after extraction, return `{"results": []}` immediately

    3. **Query first track points** (single batch query, per D-06):
       ```sql
       SELECT DISTINCT ON (flight_id) flight_id, alt, timestamp, lat, lon
       FROM research.normal_tracks
       WHERE flight_id = ANY(:flight_ids)
       ORDER BY flight_id, timestamp ASC
       ```
       Parse into dict: `first_tracks = {row["flight_id"]: row for row in query_results}`

    4. **Build output rows** (iterate over all input flight_ids):
       For each flight_id:
       - Start with upstream passthrough: `row = dict(upstream_by_id.get(flight_id, {"flight_id": flight_id}))`
       - If flight_id NOT in first_tracks: set `no_recorded_takeoff=False`, `deviation_score=0.0`, `diagnostic="insufficient history"`, `first_alt_ft=None`
       - If flight_id in first_tracks AND first_tracks[flight_id]["alt"] >= altitude_threshold_ft: set `no_recorded_takeoff=True`, `deviation_score=1.0`, `diagnostic="no anomalies found"`, `first_alt_ft=first_tracks[flight_id]["alt"]`
       - If flight_id in first_tracks AND first_tracks[flight_id]["alt"] < altitude_threshold_ft: set `no_recorded_takeoff=False`, `deviation_score=0.0`, `diagnostic="no anomalies found"`, `first_alt_ft=first_tracks[flight_id]["alt"]`

    5. **Historical enrichment for flagged flights only** (per D-11, D-12):
       - Collect callsigns of flagged flights: `flagged_callsigns = [row["callsign"] for row in output_rows if row.get("no_recorded_takeoff") and row.get("callsign")]`
       - Compute lookback_seconds from time_mode/lookback_days/start_time/end_time params (use same pattern as other behavioral cubes: `time_mode = inputs.get("time_mode") or "lookback"`, if lookback: `lookback_seconds = int(float(inputs.get("lookback_days") or 7) * 86400)`)
       - Call `history = await get_callsign_history(flagged_callsigns, lookback_seconds)`
       - For each callsign's historical flights, query first track altitudes using the same DISTINCT ON query but for the historical flight_ids
       - Compute median of historical first altitudes as `typical_first_alt_ft` (use `statistics.median()`)
       - Add `typical_first_alt_ft` to each flagged row matching that callsign; set to None if callsign has no historical data

    6. **Return:** `{"results": output_rows}`

    **Important implementation details:**
    - Use `from app.cubes.utils.historical_query import get_callsign_history` for enrichment
    - Use `from app.cubes.utils.time_utils import TIME_MODE_PARAMS` for input params
    - Use `from app.database import engine` and `sqlalchemy.text()` for queries (same as DarkFlightDetector)
    - Import `statistics` stdlib for `statistics.median()`
    - Add `import logging` and create `logger = logging.getLogger(__name__)`
    - The "insufficient history" diagnostic means zero rows in normal_tracks for that flight_id, NOT zero historical callsign data
    - `typical_first_alt_ft` should be `None` (not absent) when callsign has no historical flights, and should NOT be present at all on clean (unflagged) rows
