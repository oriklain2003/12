    **Step 2: Create `backend/app/agents/validation.py`** (per D-02, D-03, D-04, D-14):

    Implement `validate_graph(graph: WorkflowGraph, registry: CubeRegistry) -> ValidationResponse` with these checks in order:

    1. **Cycle check** — call `topological_sort(graph.nodes, graph.edges)`, catch `ValueError`. If cycle found, return early with single error issue: `rule="cycle"`, `message="Workflow contains a cycle. Circular connections are not allowed."`, `node_id=None`, `cube_name=None`, `param_name=None`.

    2. **Unknown cube check** — for each node, call `registry.get(node.data.cube_id)`. If None, append error: `rule="unknown_cube"`, `message=f"Cube '{node.data.cube_id}' is not registered in the catalog. It cannot be validated or executed."`, skip further checks on that node.

    3. **Missing required params** (per D-02) — for each node with a known cube, iterate `defn.inputs` where `param.required is True`. Build `connected_inputs = {e.targetHandle for e in graph.edges if e.target == node.id and e.targetHandle}`. A param is satisfied if: `node.data.params.get(param.name) not in (None, "", [])` OR `param.name in connected_inputs`. If unsatisfied, append error: `rule="missing_required_param"`, `message=f"Cube '{defn.name}' is missing required input '{param.name}'. Connect a data source or enter a value."`.

    4. **Dangling handles** (per D-04) — for each edge:
       - Source side: get source node's cube, build `valid_outputs = {p.name for p in defn.outputs}`, add `"__full_result__"` to valid_outputs. If `edge.sourceHandle not in valid_outputs`, append error: `rule="dangling_source_handle"`, `message=f"Cube '{defn.name}' has a connection from unknown output '{edge.sourceHandle}'. Remove this edge or update the cube."`.
       - Target side: get target node's cube, build `valid_inputs = {p.name for p in defn.inputs}`. If `edge.targetHandle not in valid_inputs`, append error: `rule="dangling_target_handle"`, `message=f"Cube '{defn.name}' has a connection to unknown input '{edge.targetHandle}'. Remove this edge or update the cube."`.

    5. **Type mismatch warnings** (per D-03) — for each edge with both handles present and sourceHandle != `"__full_result__"`, look up the source output param and target input param. If both found and `src_param.type != tgt_param.type`, append warning: `rule="type_mismatch"`, `message=f"'{src_cube_name}.{edge.sourceHandle}' ({src_param.type.value}) is connected to '{tgt_cube_name}.{edge.targetHandle}' ({tgt_param.type.value}). Types differ -- connection is allowed but may produce unexpected results."`.

    6. **Orphan node warnings** (per D-03) — build `all_connected_nodes` from all edge sources and targets. For each node not in that set (when `len(graph.nodes) > 1`), append warning: `rule="orphan_node"`, `message=f"Cube '{cube_name}' has no connections. It will not execute in this workflow."`.
