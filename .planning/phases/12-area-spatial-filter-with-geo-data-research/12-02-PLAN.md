**Step 4: Create `backend/app/geo/loader.py`** — shared loader utilities:

```python
import json
import pathlib
from shapely.geometry import shape
import shapely

_DATA_DIR = pathlib.Path(__file__).parent / "data"

def load_geojson(filename: str) -> list[dict]:
    """Load a GeoJSON FeatureCollection from the data/ directory.
    Returns the list of features.
    """
    path = _DATA_DIR / filename
    with open(path, encoding="utf-8") as f:
        fc = json.load(f)
    return fc["features"]

def build_polygon_index(features: list[dict], name_property: str) -> dict[str, object]:
    """Build a dict of name -> prepared Shapely geometry from GeoJSON features.
    Preparing geometries at load time amortizes GEOS index cost.
    Handles both Polygon and MultiPolygon geometry types.
    """
    index = {}
    for feature in features:
        name = feature["properties"].get(name_property)
        if name is None:
            continue
        geom = shape(feature["geometry"])
        shapely.prepare(geom)
        index[name] = geom
    return index
```

**Create `backend/app/geo/country_loader.py`:**

Module-level initialization: load countries.geojson once at import, build two indexes:
- `_COUNTRY_BY_NAME`: country name -> prepared Shapely geometry (property key: `ADMIN` or `name` — inspect the actual file to determine the correct property)
- `_COUNTRY_BY_ISO3`: ISO 3166 Alpha-3 code -> prepared Shapely geometry (property key: `ISO_A3` or similar)

Also build a flat list of (name, iso3, geometry) tuples for point lookup.

Expose functions:
- `classify_point(lat: float, lon: float) -> dict | None` — returns `{"country": name, "iso3": code}` or None if in ocean/unclaimed. Uses `shapely.contains_xy(geom, lon, lat)` — note lon/lat order for Shapely.
- `get_country_polygon(name_or_iso3: str) -> shapely.Geometry | None` — returns prepared polygon for a country.
- `list_countries() -> list[str]` — returns all available country names.

Performance: iterate through all country polygons for classify_point. For 255 countries with prepared geometries, this is fast enough (<10ms per point). If future profiling shows this is slow, an STRtree spatial index can be added later.

**Create `backend/app/geo/fir_loader.py`:**

Module-level initialization: load fir_uir_europe.geojson once at import. Inspect feature properties to find the FIR designator key (likely `id`, `designator`, or `name`).

Build `_FIR_INDEX`: designator -> prepared Shapely geometry.

Expose functions:
- `classify_point(lat: float, lon: float) -> dict | None` — returns `{"fir": designator, "name": fir_name}` or None if outside European FIR coverage.
- `get_fir_polygon(designator: str) -> shapely.Geometry | None`
- `list_firs() -> list[str]` — returns all FIR designators.

**Create `backend/app/geo/land_water_loader.py`:**

Module-level initialization: load ne_50m_land.geojson once at import. Land features are typically anonymous polygons (no name property). Build a list of prepared Shapely land geometries. For performance with many large polygons, use `shapely.STRtree` to build a spatial index.

Expose functions:
- `is_land(lat: float, lon: float) -> bool` — True if point is on land, False if water. Use STRtree.query for efficient lookup, then confirm with contains_xy.
- `classify_point(lat: float, lon: float) -> str` — returns "land" or "water".

**Important for all loaders:**
- Use `shapely.contains_xy(geom, lon, lat)` — Shapely uses (x=lon, y=lat) order.
- Log a warning if the GeoJSON file fails to load (don't crash the entire backend).
- Wrap module-level loading in a try/except — if data files are missing, loaders should degrade gracefully with logged warnings and return None/empty results.
