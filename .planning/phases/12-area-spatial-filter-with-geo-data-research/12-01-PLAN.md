  <action>
Create `AreaSpatialFilterCube` in `backend/app/cubes/area_spatial_filter.py` following the SquawkFilter dual-provider template. Category: `CubeCategory.FILTER`.

**Input params:**
- `full_result` (JSON_OBJECT, required=False, accepts_full_result=True) — upstream bundle with flight_ids/hex_list
- `flight_ids` (LIST_OF_STRINGS, required=False) — direct flight IDs for FR
- `hex_list` (LIST_OF_STRINGS, required=False) — direct hex list for Alison
- `provider` (STRING, required=False, default="fr", widget_hint="select") — "fr" or "alison"
- `polygon` (JSON_OBJECT, required=False, widget_hint="polygon") — manual polygon [[lat, lon], ...] or from upstream connection
- `altitude_threshold` (NUMBER, required=False, default=1000) — feet; below this = potential landing/takeoff
- `lookback_hours` (NUMBER, required=False, default=24) — Alison time window (prevents full scan of 46M-row positions table)

**Output params:**
- `flight_ids` (LIST_OF_STRINGS) — IDs of flights confirmed inside polygon (flight_id for FR, hex for Alison)
- `hex_list` (LIST_OF_STRINGS) — same list aliased as hex_list for downstream Alison consumers
- `count` (NUMBER) — number of matching flights

**Execute method flow:**

1. **Extract IDs** — dual-provider pattern from SquawkFilter (full_result fallback to direct inputs). Empty guard returns `{"flight_ids": [], "hex_list": [], "count": 0}`.

2. **Parse polygon** — from inputs. If polygon is a list of [lat, lon] pairs, use directly. If None/empty, return empty with warning log.

3. **Build Shapely polygon** — `Polygon([(lon, lat) for lat, lon in polygon])` — CRITICAL: swap lat/lon to lon/lat for Shapely. Call `shapely.prepare(poly_shapely)`.

4. **Compute bbox** — min/max lat and lon from polygon coordinates for SQL pre-filter.

5. **Query positions by provider:**
   - **FR:** Query `research.normal_tracks` with `flight_id = ANY(:ids)` + bbox filter + `lat IS NOT NULL AND lon IS NOT NULL`. Columns: `flight_id, timestamp, lat, lon, alt, vspeed`. ORDER BY `flight_id, timestamp`. LIMIT 200000.
   - **Alison:** Query `public.positions` with `hex = ANY(:ids)` + `ts >= to_timestamp(:cutoff)` + bbox filter. Columns: `hex, ts, lat, lon, alt_baro, baro_rate, on_ground`. ORDER BY `hex, ts`. LIMIT 200000. Compute cutoff: `int(time.time() - lookback_hours * 3600)`. NOTE: Do NOT add `lat IS NOT NULL` filter for Alison — on_ground=True rows with null lat/lon are needed for movement classification; filter nulls only in the Python PIP check.

6. **Group positions by flight/hex** — dict of id -> list of position dicts, preserving temporal order.

7. **For each flight, find polygon-inside positions:**
   - Skip positions where lat or lon is None (but keep them in the full sequence for on_ground classification).
   - Use `shapely.contains_xy(poly_shapely, float(lon), float(lat))` for each position with valid coordinates.
   - Track which positions are inside and which are outside.

8. **Compute entry/exit times and duration for flights with positions inside:**
   - `entry_time` = timestamp of first position inside polygon
   - `exit_time` = timestamp of last position inside polygon
   - `time_in_area` = exit_time - entry_time (as seconds for computation, formatted as human-readable string in output)
   - Normalize timestamps to ISO 8601: FR `datetime.utcfromtimestamp(row["timestamp"]).isoformat() + "Z"`, Alison `row["ts"].isoformat()`

9. **Classify movement** for each matching flight:

   **Alison classification** (`classify_movement_alison`):
   - Use ALL positions in temporal order (including null-coordinate ones) for on_ground analysis.
   - Priority 1: Check `on_ground` transitions in the positions that overlap the polygon time window (entry to exit). `False -> True` = landing, `True -> False` = takeoff.
   - Priority 2: Fall back to altitude + baro_rate. If all `alt_baro` values in area < altitude_threshold and average `baro_rate` < -300 = landing, > 300 = takeoff.
   - Default: "cruise"
   - Handle case where `on_ground=True` has no lat/lon: use last airborne position as spatial landing point.

   **FR classification** (`classify_movement_fr`):
   - No `on_ground` column — infer from `alt` + `vspeed`.
   - If average `alt` in polygon < altitude_threshold: average `vspeed` < -300 ft/min = landing, > 300 ft/min = takeoff.
   - Default: "cruise"
   - Handle missing alt gracefully (default to "cruise" if no altitude data).

10. **Build output:**
    - `flight_ids`: list of matching IDs
    - `hex_list`: same list (for Alison downstream compatibility)
    - `count`: len of matching IDs
    - `per_flight_details`: dict of id -> {entry_time, exit_time, time_in_area, movement_classification, positions_in_area (count)} — auto-included in __full_result__

Log warnings if: LIMIT 200000 is hit (may miss positions), polygon has < 3 vertices, no IDs provided.
