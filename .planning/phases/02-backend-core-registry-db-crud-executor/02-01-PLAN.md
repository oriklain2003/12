    **Cubes Router (backend/app/routers/cubes.py):**
    Create APIRouter with prefix="/api/cubes" and tags=["cubes"]. Single endpoint: GET /catalog returns `list[CubeDefinition]` by calling `registry.catalog()`. Import registry singleton from `app.engine.registry`.

    **Workflows Router (backend/app/routers/workflows.py):**
    Create APIRouter with prefix="/api/workflows" and tags=["workflows"]. Five CRUD endpoints:

    1. `POST /` (create): Accept WorkflowCreate body. Create Workflow ORM instance, set name and graph_json (call `body.graph_json.model_dump()` to serialize WorkflowGraph to dict for JSONB). `session.add()`, `await session.commit()`, `await session.refresh(wf)`. Return WorkflowResponse. Reconstruct graph_json for response using `WorkflowGraph.model_validate(wf.graph_json)`.

    2. `GET /` (list): `select(Workflow).order_by(Workflow.updated_at.desc())`. Return `list[WorkflowResponse]`. For each, validate graph_json back to WorkflowGraph.

    3. `GET /{workflow_id}` (get): Accept `workflow_id: uuid.UUID`. Select by id. If None, raise HTTPException(404). Return WorkflowResponse.

    4. `PUT /{workflow_id}` (update): Accept WorkflowUpdate body. Fetch workflow or 404. If `body.name is not None`, update name. If `body.graph_json is not None`, update graph_json (model_dump()). Set updated_at manually to `datetime.now(timezone.utc)`. Commit, refresh, return WorkflowResponse.

    5. `DELETE /{workflow_id}` (delete): Fetch or 404. `await session.delete(wf)`, commit. Return `{"detail": "Workflow deleted"}`.

    All endpoints depend on `db: AsyncSession = Depends(get_db)`.

    **Main.py Update (backend/app/main.py):**
    Import cubes_router from `app.routers.cubes` and workflows_router from `app.routers.workflows`. Add `app.include_router(cubes_router)` and `app.include_router(workflows_router)` after CORS middleware.

    **Run Alembic migration:**
    After creating the router files, run `cd backend && uv run alembic upgrade head` to create the workflows table. If the migration fails because the table already exists, that is fine -- proceed.

Since `from_attributes = True` is set and the JSONB dict maps directly to WorkflowGraph fields, Pydantic v2 will auto-validate it. If this causes issues, explicitly construct WorkflowGraph.model_validate(wf.graph_json) and build the response manually.
