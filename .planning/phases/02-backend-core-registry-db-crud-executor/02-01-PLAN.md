Create `routers/cubes.py`:
- `router = APIRouter(prefix="/api/cubes", tags=["cubes"])`
- `GET /catalog` endpoint:
  - Imports `registry` from `app.engine.registry`
  - Returns `registry.get_all_definitions()` as JSON array
  - Response model: `list[CubeDefinition]`

Create `routers/workflows.py`:
- `router = APIRouter(prefix="/api/workflows", tags=["workflows"])`
- Depends on `get_db` for async session injection
- **POST /** (create):
  - Accepts `WorkflowCreate` body
  - Creates `Workflow(name=body.name, graph_json=body.graph_json.model_dump())`
  - Returns `WorkflowResponse` with 201 status
- **GET /** (list):
  - Queries all workflows ordered by `updated_at` descending
  - Returns `list[WorkflowResponse]`
- **GET /{workflow_id}** (get):
  - `workflow_id: uuid.UUID` path param
  - Returns `WorkflowResponse` or raises 404
- **PUT /{workflow_id}** (update):
  - Accepts `WorkflowUpdate` body
  - Updates only provided fields (name, graph_json)
  - For graph_json, use `.model_dump()` before storing
  - Returns updated `WorkflowResponse` or 404
- **DELETE /{workflow_id}** (delete):
  - Deletes workflow, returns 204 (no content)
  - Raises 404 if not found

Important: When reading graph_json back from the database (JSONB), it will be a plain dict. The Pydantic `WorkflowResponse` with `from_attributes=True` needs to handle this. Use `WorkflowGraph.model_validate(workflow.graph_json)` to reconstruct the Pydantic model before returning, or ensure the response serialization handles it. One clean approach: add a property or adjust the response construction to validate the JSONB dict back into a WorkflowGraph.

Update `main.py`:
- Import both routers: `from app.routers import cubes, workflows`
- Add: `app.include_router(cubes.router)` and `app.include_router(workflows.router)`
