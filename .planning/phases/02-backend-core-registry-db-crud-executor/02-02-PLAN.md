  <action>
    Create `backend/app/engine/executor.py` with four public functions:

    **topological_sort(nodes, edges) -> list[str]:**
    Kahn's algorithm. Build in-degree map and adjacency list from edges. Process zero-in-degree nodes via deque. If processed count < total nodes, raise ValueError("Workflow graph contains a cycle"). Return ordered list of node IDs.

    **resolve_inputs(node, edges, results) -> dict[str, Any]:**
    Start with `dict(node.data.params)` (manual values). For each edge where edge.target == node.id: if edge.sourceHandle == "__full_result__", set value to entire source outputs dict (this IS the Full Result bundle -- BACK-10). Otherwise get value from `results[edge.source]["outputs"][edge.sourceHandle]`. Set `inputs[edge.targetHandle] = value`. Connections win over manual values (BACK-12).

    **apply_row_limit(outputs, limit) -> tuple[dict, bool]:**
    For each output value that is a list with len > limit, slice to `value[:limit]` and set truncated=True. Return (capped_outputs, any_truncated). Uses `settings.result_row_limit` as default limit (BACK-11).

    **async execute_graph(graph: WorkflowGraph) -> dict:**
    1. Call topological_sort -- if ValueError, raise HTTPException(400, detail=str(e))
    2. Build node_map {id: node}, results dict, failed_or_skipped set
    3. For each node_id in order:
       a. Check if any source node (from edges where target==node_id) is in failed_or_skipped. If yes, mark as `{"status": "skipped", "reason": "upstream cube failed or was skipped", "outputs": {}}` and add to failed_or_skipped
       b. Look up cube via `registry.get(node.data.cube_id)`. If None, mark as error
       c. Call resolve_inputs() to get merged inputs
       d. Try `await cube.execute(**inputs)`. On success, call apply_row_limit(), store `{"status": "done", "outputs": capped_outputs, "truncated": truncated}`
       e. On exception, store `{"status": "error", "message": str(e), "outputs": {}}`, add to failed_or_skipped
    4. Return results dict keyed by node_id

    Response shape matches the locked decision:
    ```json
    {
      "node-1": {"status": "done", "outputs": {...}, "truncated": false},
      "node-2": {"status": "error", "message": "...", "outputs": {}},
      "node-3": {"status": "skipped", "reason": "...", "outputs": {}}
    }
    ```

For execute_graph tests, create mock cubes by subclassing BaseCube and registering them temporarily (or mock the registry). Use WorkflowNode/WorkflowEdge/WorkflowGraph directly.     **AllFlights Cube (backend/app/cubes/all_flights.py):**
    Subclass BaseCube. cube_id="all_flights", name="All Flights", description="Query flight metadata from Tracer 42 database", category=CubeCategory.DATA_SOURCE.

    **Inputs (all optional):**
    - `time_range_seconds` (NUMBER): Relative time filter -- last N seconds. Default: 604800 (7 days).
    - `start_time` (STRING): Absolute start time as epoch seconds string. Overrides relative if both provided.
    - `end_time` (STRING): Absolute end time as epoch seconds string.
    - `flight_ids` (LIST_OF_STRINGS): Filter to specific flight IDs. Accepts from connection or manual entry.
    - `callsign` (STRING): Callsign filter (ILIKE pattern).
    - `min_altitude` (NUMBER): Minimum altitude in feet. Filters flights where min_altitude_ft >= this value.
    - `max_altitude` (NUMBER): Maximum altitude in feet. Filters flights where max_altitude_ft <= this value.
    - `polygon` (JSON_OBJECT): Array of [lat, lon] coordinate pairs defining a geofence boundary. Accepts as `list[list[float]]`.

    **Outputs:**
    - `flights` (JSON_OBJECT): Array of flight record objects (row-based table format per locked decision).
    - `flight_ids` (LIST_OF_STRINGS): Array of flight_id strings for downstream cubes.

    **execute() implementation:**
    1. Build parameterized SQL query against `research.flight_metadata`. Start with `SELECT flight_id, callsign, first_seen_ts, last_seen_ts, min_altitude_ft, max_altitude_ft, origin_airport, destination_airport, is_anomaly, is_military, start_lat, start_lon, end_lat, end_lon FROM research.flight_metadata WHERE 1=1`.
    2. Apply filters conditionally:
       - Time: If start_time and end_time provided, use absolute range (`first_seen_ts <= end_epoch AND last_seen_ts >= start_epoch`). Otherwise use relative: `cutoff = int(time.time()) - time_range_seconds`, filter `last_seen_ts >= cutoff`.
       - flight_ids: `WHERE flight_id = ANY($N)` (asyncpg parameterized)
       - callsign: `WHERE callsign ILIKE $N` (wrap with % wildcards)
       - altitude: `WHERE min_altitude_ft >= $N AND max_altitude_ft <= $N`
    3. Add `LIMIT 5000` to SQL query as safety cap before polygon filtering.
    4. Execute query using `engine` from `app.database` -- create an async connection via `engine.connect()`, use `await conn.execute(text(sql), params)`. Use SQLAlchemy `text()` for raw SQL with `:param` style bindings.
    5. If polygon input provided (non-None, non-empty list):
       a. Get candidate flight_ids from step 4 results
       b. Query `research.normal_tracks` for those flight_ids: `SELECT DISTINCT flight_id, lat, lon FROM research.normal_tracks WHERE flight_id = ANY(:ids)`
       c. Apply point_in_polygon ray-casting filter in Python: keep flights that had ANY track point inside the polygon
       d. Filter the results from step 4 to only include flights that passed polygon check
    6. Convert result rows to list of dicts (array-of-objects format per locked decision). Each dict has same keys.
    7. Return `{"flights": rows_as_dicts, "flight_ids": [row["flight_id"] for row in rows_as_dicts]}`.

    **point_in_polygon helper (inside all_flights.py):**
    Pure Python ray-casting algorithm. Takes lat, lon, and polygon (list of [lat, lon] pairs). Returns bool. This is needed because PostGIS is NOT installed on the Tracer 42 RDS instance (confirmed in research).

    NOTE: The cube acquires its own database connection (not from FastAPI dependency injection). Use `async with engine.connect() as conn:` for direct SQL. This keeps BaseCube.execute() signature clean (**inputs only) per research guidance.
