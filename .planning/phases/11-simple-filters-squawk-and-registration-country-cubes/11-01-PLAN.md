  <action>
Create `backend/app/cubes/alison_flights.py` following the AllFlights pattern exactly but targeting `public.aircraft` joined with `public.positions`.

**Class:** `AlisonFlightsCube(BaseCube)`
- `cube_id = "alison_flights"`
- `name = "Alison Flights"`
- `description = "Query aircraft from the Alison provider (public schema)"`
- `category = CubeCategory.DATA_SOURCE`

**Inputs** (mirror AllFlights where applicable):
- `time_range_seconds` (NUMBER, default=604800, widget_hint="relative_time") -- relative time filter on positions.ts
- `start_time` (STRING, widget_hint="datetime") -- absolute start as epoch seconds string
- `end_time` (STRING, widget_hint="datetime") -- absolute end as epoch seconds string
- `hex_filter` (LIST_OF_STRINGS) -- filter to specific ICAO24 hex addresses
- `callsign` (STRING) -- ILIKE pattern match on positions.flight
- `aircraft_type` (STRING) -- ILIKE match on aircraft.icao_type
- `min_altitude` (NUMBER) -- min alt_baro filter
- `max_altitude` (NUMBER) -- max alt_baro filter
- `polygon` (JSON_OBJECT, widget_hint="polygon") -- geofence boundary [[lat, lon], ...]
- `min_lat`, `max_lat`, `min_lon`, `max_lon` (NUMBER) -- bbox filters

**Outputs:**
- `flights` (JSON_OBJECT) -- array of aircraft record objects
- `hex_list` (LIST_OF_STRINGS) -- ICAO24 hex addresses for downstream cubes

**execute() logic:**

1. Build parameterized SQL query:
   ```sql
   SELECT a.hex, a.registration, a.icao_type, a.type_description, a.category,
          array_agg(DISTINCT p.flight) FILTER (WHERE p.flight IS NOT NULL) AS callsigns,
          MIN(p.ts) AS first_seen_ts, MAX(p.ts) AS last_seen_ts,
          MIN(p.alt_baro) AS min_alt_baro, MAX(p.alt_baro) AS max_alt_baro
   FROM public.aircraft a
   JOIN public.positions p ON a.hex = p.hex
   WHERE 1=1
   {dynamic filters}
   GROUP BY a.hex, a.registration, a.icao_type, a.type_description, a.category
   LIMIT 5000
   ```

2. Time filter: Same logic as AllFlights -- if start_time + end_time provided, use `p.ts BETWEEN :start_epoch AND :end_epoch` (convert epoch seconds string to integer). Otherwise use relative cutoff on p.ts.

3. Other filters: hex_filter (`a.hex = ANY(:hex_filter)`), callsign (`p.flight ILIKE :callsign`), aircraft_type (`a.icao_type ILIKE :aircraft_type`), altitude (`p.alt_baro >= :min_altitude` / `<= :max_altitude`), bbox (on p.lat, p.lon).

4. Polygon filter: If polygon provided with >= 3 points, derive bbox for SQL pre-filter, then Python ray-casting post-filter using `point_in_polygon` imported from `app.cubes.all_flights`. For polygon, need a second query to get per-hex positions within bbox, then check if any point is inside polygon. Only include hexes that have at least one position inside the polygon.

5. Output: `{"flights": rows, "hex_list": [row["hex"] for row in rows]}`

6. Include debug logging of the final SQL (same pattern as AllFlights).

IMPORTANT: Use `array_agg(DISTINCT p.flight)` for callsigns so we get one row per hex (per RESEARCH.md open question 4 recommendation).
