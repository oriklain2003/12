Create `backend/app/cubes/registration_country_filter.py` following the FilterFlights pattern.

**Class:** `RegistrationCountryFilterCube(BaseCube)`
- `cube_id = "registration_country_filter"`
- `name = "Registration Country Filter"`
- `description = "Filter aircraft by registration country via ICAO24 hex prefix and tail number prefix. Alison provider only."`
- `category = CubeCategory.FILTER`

**Inputs:**
- `full_result` (JSON_OBJECT, accepts_full_result=True) -- Full result from Alison Flights
- `hex_list` (LIST_OF_STRINGS) -- Direct hex list fallback
- `filter_mode` (STRING, default="include", widget_hint="select") -- "include" (keep matching) or "exclude" (remove matching)
- `countries` (LIST_OF_STRINGS, widget_hint="tags") -- Country names to match (e.g., ["Iran", "Syria"])
- `regions` (LIST_OF_STRINGS, widget_hint="tags") -- Region group tags to expand (e.g., ["black", "gray"])

**Outputs:**
- `flight_ids` (LIST_OF_STRINGS) -- Hex addresses that passed the filter
- `count` (NUMBER) -- Number of passing aircraft

**execute() logic:**

1. **Extract hex_list from inputs:**
   - If `full_result` provided: extract `hex_list` key
   - Fallback to direct `hex_list` input
   - Empty guard: if no hex_list, return `{"flight_ids": [], "count": 0}`

2. **Build target country set:**
   - Start with `countries` input (direct country names)
   - Expand `regions` input using `expand_regions()` from icao24_lookup
   - Union both sets. If empty (no countries, no regions), return all hexes (no filter applied) with a warning log.

3. **Resolve country for each hex -- primary check (ICAO24 range):**
   - For each hex in hex_list, call `resolve_country_from_hex(hex)` from icao24_lookup
   - Build dict: `resolved_countries: dict[str, dict]` mapping hex to `{"country": str | None, "match_type": "hex_range" | "tail_prefix" | "both" | "unknown"}`

4. **Secondary check (tail number prefix):**
   - Query `public.aircraft` for registration numbers: `SELECT hex, registration FROM public.aircraft WHERE hex = ANY(:hex_list)`
   - For each hex, call `resolve_country_from_registration(registration)` from icao24_lookup
   - If hex had no country from ICAO24 range but has one from tail prefix: set country and match_type="tail_prefix"
   - If hex has country from BOTH: set match_type="both" (keep the ICAO24 range result as primary)

5. **Apply filter_mode:**
   - Build set of each hex's resolved country (may be None for unknown)
   - `include` mode: keep hex if its country is in target_countries set
   - `exclude` mode: keep hex if its country is NOT in target_countries set
   - Hexes with country=None (unresolvable): EXCLUDE in include mode, KEEP in exclude mode (conservative -- unknown aircraft are not excluded)

6. **Return:**
   ```python
   return {
       "flight_ids": list(passing_hexes),
       "count": len(passing_hexes),
       "country_details": resolved_countries,  # included in __full_result__
   }
   ```
   The `country_details` dict provides per-hex resolution info for downstream cubes / Full Result display.

IMPORTANT: Use `from app.cubes.icao24_lookup import resolve_country_from_hex, resolve_country_from_registration, expand_regions` -- this module was created in Plan 01.
IMPORTANT: Unresolvable hexes MUST still appear in country_details with `country: null` per RESEARCH.md anti-pattern guidance.
