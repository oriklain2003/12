**Class:** `SquawkFilterCube(BaseCube)`
- `cube_id = "squawk_filter"`
- `name = "Squawk Filter"`
- `description = "Filter flights by transponder squawk codes with code-change detection. Supports FR and Alison providers."`
- `category = CubeCategory.FILTER`

**Inputs:**
- `full_result` (JSON_OBJECT, accepts_full_result=True) -- Full result bundle from upstream cube
- `flight_ids` (LIST_OF_STRINGS) -- Direct flight_id list (FR provider fallback)
- `hex_list` (LIST_OF_STRINGS) -- Direct hex list (Alison provider fallback)
- `provider` (STRING, default="fr", widget_hint="select") -- Provider: "fr" or "alison". Description should mention the options.
- `mode` (STRING, default="custom", widget_hint="select") -- Mode: "custom" (user codes) or "emergency" (preset)
- `squawk_codes` (LIST_OF_STRINGS) -- Target squawk codes for custom mode (e.g., ["7500", "7700"])
- `lookback_hours` (NUMBER, default=24) -- Time window safety cap for Alison positions queries (hours). Required to avoid scanning all 46M rows.

**Outputs:**
- `flight_ids` (LIST_OF_STRINGS) -- IDs of flights with matching squawk (flight_id for FR, hex for Alison)
- `count` (NUMBER) -- Number of matching flights

**execute() logic:**

1. **Extract identifiers from inputs:**
   - If `full_result` provided: extract `flight_ids` or `hex_list` key depending on provider
   - Fallback to direct `flight_ids` (FR) or `hex_list` (Alison) input
   - Empty guard: if no identifiers, return `{"flight_ids": [], "count": 0}`

2. **Determine target codes based on mode:**
   - `custom`: use `squawk_codes` input directly. If empty, return empty result.
   - `emergency` with FR provider: target codes = ["7500", "7600", "7700"]
   - `emergency` with Alison provider: use `positions.emergency` column instead (see step 3)

3. **Query squawk history by provider:**

   **FR provider:**
   ```sql
   SELECT flight_id AS id, squawk, timestamp AS ts
   FROM research.normal_tracks
   WHERE flight_id = ANY(:ids) AND squawk IS NOT NULL
   ORDER BY flight_id, timestamp
   ```

   **Alison provider (custom mode):**
   ```sql
   SELECT hex AS id, squawk, emergency, ts
   FROM public.positions
   WHERE hex = ANY(:ids)
     AND ts >= NOW() - INTERVAL ':lookback hours'
     AND squawk IS NOT NULL
   ORDER BY hex, ts
   ```
   Use parameterized interval: `ts >= NOW() - make_interval(hours => :lookback)` or compute the cutoff timestamp in Python and pass as parameter.

   **Alison provider (emergency mode):**
   ```sql
   SELECT hex AS id, squawk, emergency, ts
   FROM public.positions
   WHERE hex = ANY(:ids)
     AND ts >= :cutoff_ts
     AND emergency IS NOT NULL AND emergency != 'none'
   ORDER BY hex, ts
   ```

4. **Matching logic:**
   - Group rows by identifier (flight_id or hex)
   - For custom mode: a flight passes if ANY row has `squawk` in target codes set
   - For emergency mode with Alison: a flight passes if ANY row has `emergency != 'none'`
   - For emergency mode with FR: a flight passes if ANY row has squawk in ["7500", "7600", "7700"]
   - Use string comparison for squawk values (cast input codes to str if needed, based on Task 1 findings)

5. **Code-change detection (for Full Result):**
   - For ALL matching flights (not just emergency), process squawk history ordered by timestamp
   - Track previous squawk code per flight
   - When code changes (prev is not None and prev != current), record: `{"from": prev, "to": current, "ts": timestamp}`
   - Per-flight detail: `{"codes_seen": [...], "code_changes": [{"from": "...", "to": "...", "ts": ...}], "matched_codes": [...]}`
   - For Alison emergency mode, also include the `emergency` field values in details

6. **Return:**
   ```python
   return {
       "flight_ids": list(passing_ids),
       "count": len(passing_ids),
       # Full Result auto-appended by BaseCube includes per_flight_details
   }
   ```
   Store per_flight_details in the return dict so it appears in __full_result__.

IMPORTANT: Squawk codes may be stored as strings in the DB. Use string comparison throughout. Cast any integer input to string before comparing.
IMPORTANT: Always include a LIMIT on the positions query (e.g., 100000 rows) as a safety net.
IMPORTANT: Compute the lookback cutoff in Python using `time.time() - lookback_hours * 3600` and pass as integer to avoid SQL injection with interval syntax.
