  <action>
Create `backend/app/cubes/filter_flights.py` implementing FilterFlightsCube (category: FILTER):

**Inputs** (all optional):
- `full_result` (JSON_OBJECT, accepts_full_result=True) — receives AllFlights __full_result__ bundle
- `flight_ids` (LIST_OF_STRINGS) — direct flight_ids fallback if no full_result
- `max_altitude_ft` (NUMBER) — exclude flights with any track point above this
- `min_speed_knots` (NUMBER) — exclude flights that never reach this speed (check MAX(gspeed) >= threshold)
- `max_speed_knots` (NUMBER) — exclude flights that exceed this speed (check MAX(gspeed) <= threshold)
- `min_duration_minutes` (NUMBER) — exclude flights shorter than this
- `max_duration_minutes` (NUMBER) — exclude flights longer than this
- `polygon` (JSON_OBJECT, widget_hint="polygon") — geofence [[lat,lon],...] array

**Outputs:**
- `filtered_flight_ids` (LIST_OF_STRINGS)
- `filtered_flights` (JSON_OBJECT)

**execute() logic — two-tier approach per CONTEXT.md locked decisions:**

1. Extract flights + flight_ids from full_result dict (keys: "flights", "flight_ids"). Fallback to direct flight_ids input if no full_result. Guard empty flight_ids early — return empty results.

2. **Tier 1 (metadata — no DB query):** If min/max_duration_minutes set AND flights metadata available, compute duration as `(last_seen_ts - first_seen_ts) / 60.0` for each flight. Exclude flights outside range. Flights missing timestamps are excluded. Update surviving flight_ids set.

3. Early exit if no survivors after Tier 1.

4. **Tier 2 (normal_tracks — SQL aggregate):** If any of max_altitude_ft, min_speed_knots, max_speed_knots are set, run single query:
   ```sql
   SELECT flight_id, MAX(alt) AS max_alt, MAX(gspeed) AS max_speed
   FROM research.normal_tracks
   WHERE flight_id = ANY(:flight_ids)
   GROUP BY flight_id
   ```
   For each surviving flight_id:
   - If not in track_stats (no track data) → exclude
   - If max_altitude_ft set and (max_alt or 0) > max_altitude_ft → exclude
   - If min_speed_knots set and (max_speed or 0) < min_speed_knots → exclude
   - If max_speed_knots set and (max_speed or 0) > max_speed_knots → exclude

   If NO altitude/speed filters are active, still query normal_tracks to confirm flights have track data (exclude those without). Use a simpler query: `SELECT DISTINCT flight_id FROM research.normal_tracks WHERE flight_id = ANY(:ids)`.

5. **Polygon filter (if polygon with >= 3 points and passing_ids not empty):**
   Import `point_in_polygon` from `app.cubes.all_flights`. Query track points with bounding box SQL pre-filter. For each track point, check if inside polygon. Flight passes if ANY point is inside (early-exit per flight). Intersect passing_ids with flights_in_polygon.

6. Build output: filter the original flights metadata list down to passing flight_ids. Return `{"filtered_flight_ids": [...], "filtered_flights": [...]}`.

**Important implementation details:**
- Use `from app.cubes.all_flights import point_in_polygon` (per RESEARCH.md recommendation)
- Use `engine.connect()` directly (not FastAPI DI) — same pattern as all other cubes
- Use `(stats["max_alt"] or 0)` for NULL-safe comparisons
- Add logging with `logger = logging.getLogger(__name__)` — log query before execution
- No LIMIT on GROUP BY query (returns 1 row per flight, bounded by input count)
