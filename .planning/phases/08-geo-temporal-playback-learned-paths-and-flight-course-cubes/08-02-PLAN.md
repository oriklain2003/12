  <action>
Create `backend/app/cubes/get_learned_paths.py` implementing `GetLearnedPathsCube(BaseCube)`.

**Class attributes:**
- `cube_id = "get_learned_paths"`
- `name = "Get Learned Paths"`
- `description = "Query learned flight paths with optional filtering and corridor generation"`
- `category = CubeCategory.DATA_SOURCE`

**Inputs (all optional):**
- `origin` (string) — origin airport filter
- `destination` (string) — destination airport filter
- `path_id` (string) — specific path ID filter
- `polygon` (json_object, widget_hint="polygon") — geofence boundary (array of [lat, lon] pairs)
- `min_member_count` (number) — minimum member count filter
- `output_geometry` (string, default="centerline") — `"centerline"` (LineString) or `"corridor"` (buffered Polygon)
- `width_override` (number, optional) — overrides DB width_nm for corridor generation

**Outputs:**
- `paths` (json_object) — array of path rows with geometry
- `path_ids` (list_of_strings) — list of path IDs

**execute() logic:**

1. Build parameterized SQL against `public.learned_paths` (adapt column names based on Task 1 schema inspection). Apply filters for origin, destination, path_id, min_member_count via SQL WHERE clauses.

2. Fetch rows via `engine.connect()`.

3. **Polygon filter (if provided):** Import `point_in_polygon` from `app.cubes.all_flights`. For each path, test if any of its centerline points fall inside the polygon. Keep paths where at least one point is inside. This follows the same pattern as AllFlights.

4. **Geometry generation per row:**
   - Parse the centerline from the DB column (the format depends on Task 1 inspection — likely an array of coordinate pairs or a JSONB structure).
   - **Centerline mode:** Build GeoJSON LineString: `{"type": "LineString", "coordinates": [[lon, lat], ...]}`. Remember GeoJSON uses [lon, lat] order.
   - **Corridor mode:** Use Shapely to buffer the centerline:
     ```python
     from shapely.geometry import LineString, mapping
     line = LineString([(lon, lat) for lat, lon in centerline_coords])
     width_nm = width_override if width_override else row_width_nm
     buffer_deg = (width_nm / 2.0) / 60.0  # 1 NM ~= 1/60 degree
     polygon_geom = line.buffer(buffer_deg, cap_style='flat', join_style='round')
     geometry = dict(mapping(polygon_geom))
     ```

5. Build output rows: `{id, origin, destination, geometry, width_nm, member_count}` per path. Return `{"paths": output_rows, "path_ids": [row["id"] for row in output_rows]}`.

**Edge cases:**
- Paths with fewer than 2 centerline points: skip (cannot form LineString or corridor)
- Empty polygon input: skip polygon filtering
- No results: return empty arrays

**Coordinate order convention:** Document at top of file: "DB stores [lat, lon] internally; GeoJSON output uses [lon, lat]; Shapely uses (x=lon, y=lat)"
