- `geometry_column` (string, required, default="geometry") — column containing GeoJSON geometry
- `timestamp_column` (string, required, default="timestamp") — column containing timestamp values
- `id_column` (string, optional) — column to identify distinct objects (for color assignment)
- `color_by_column` (string, optional) — column for grouping colors (overrides id_column for coloring)

**Outputs:**
- `data` (json_object) — passthrough of input data (this is visualization-only)

**execute() logic:**
This is a passthrough cube. The execute method simply returns the input data unchanged:
```python
async def execute(self, **inputs):
    data = inputs.get("data", [])
    # Passthrough — visualization happens on frontend
    return {"data": data}
```

The cube exists so users can configure visualization parameters (geometry_column, timestamp_column, etc.) on the node, and the frontend uses these params plus the `widget="geo_playback"` field to render the GeoPlaybackWidget.

Modify `frontend/src/components/Results/ResultsDrawer.tsx` to dispatch to custom visualization components based on the selected cube's `widget` field.

1. **Add a Zustand selector** to read the selected cube's widget and params:
```typescript
const cubeWidget = useFlowStore((s) => {
  const node = s.nodes.find((n) => n.id === s.selectedResultNodeId);
  return node?.data.cubeDef.widget ?? null;
});

const cubeParams = useFlowStore((s) => {
  const node = s.nodes.find((n) => n.id === s.selectedResultNodeId);
  return node?.data.params ?? {};
});
```

2. **Add a placeholder component** for the GeoPlaybackWidget (the real implementation comes in Plan 04). Create an inline placeholder that shows "Geo Playback Widget loading..." so the dispatch logic can be verified without the full widget:
```typescript
// Placeholder until Plan 04 implements the real GeoPlaybackWidget
function GeoPlaybackPlaceholder({ rows, params }: { rows: unknown[]; params: Record<string, unknown> }) {
  return (
    <div style={{ display: 'flex', alignItems: 'center', justifyContent: 'center', height: '100%', color: 'var(--text-secondary)' }}>
      <div>
        <p>Geo Playback Widget</p>
        <p style={{ fontSize: '0.8rem' }}>{rows.length} rows | geometry: {String(params.geometry_column ?? 'geometry')} | timestamp: {String(params.timestamp_column ?? 'timestamp')}</p>
      </div>
    </div>
  );
}
```

3. **Modify the right panel rendering logic.** The current pattern only shows ResultsMap when geoInfo is detected. Replace with a three-way decision:
   - If `cubeWidget === 'geo_playback'` → render GeoPlaybackPlaceholder (later replaced by real widget) with ResizeDivider
   - Else if `geoInfo` → render ResultsMap (existing behavior)
   - Else → no right panel (existing behavior)

The key change is in the JSX — replace the `{geoInfo && (...)}` block with:
```tsx
{(cubeWidget || geoInfo) && (
  <>
    <ResizeDivider containerRef={contentRef} onSplitChange={setSplitRatio} />
    <div className="results-drawer__map-pane" style={{ flex: 1 }}>
      {cubeWidget === 'geo_playback' ? (
        <GeoPlaybackPlaceholder rows={results.rows} params={cubeParams} />
      ) : geoInfo ? (
        <ResultsMap
          rows={results.rows}
          geoInfo={geoInfo}
          selectedRowIndex={selectedRowIndex}
          onMarkerClick={setSelectedRowIndex}
        />
      ) : null}
    </div>
  </>
)}
```

Also update the table pane flex style to account for widget panels:
```tsx
style={{ flex: (cubeWidget || geoInfo) ? `0 0 ${splitRatio * 100}%` : '1' }}
```

This makes the dispatch extensible — future widget types just add another branch to the conditional.
