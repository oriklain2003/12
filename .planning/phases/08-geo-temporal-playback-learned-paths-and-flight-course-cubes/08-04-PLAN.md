Create `frontend/src/components/Visualization/` directory and two files.

**GeoPlaybackWidget.tsx** — Props and state:

```typescript
interface GeoPlaybackWidgetProps {
  rows: unknown[];
  params: Record<string, unknown>;
}
```

Extract from params: `geometry_column` (default "geometry"), `timestamp_column` (default "timestamp"), `id_column`, `color_by_column`.

**1. Timestamp extraction and range computation (useMemo):**
- Parse all timestamps from `rows[timestampCol]` as numbers
- Compute `tsMin`, `tsMax` from the data
- Guard: if `tsMin === tsMax`, set `tsMax = tsMin + 1` to avoid division by zero
- Initialize `windowStart` and `windowEnd` state: start at tsMin, end at tsMin + (tsMax-tsMin)*0.05 (5% window)

**2. Color assignment (useMemo):**
- Hardcoded categorical palette (D3 Tableau10 colors, no D3 dependency):
  ```typescript
  const COLORS = ['#4e79a7','#f28e2b','#e15759','#76b7b2','#59a14f','#edc948','#b07aa1','#ff9da7','#9c755f','#bab0ac'];
  ```
- Build a `Map<string, string>` keyed by `color_by_column ?? id_column` value
- Assign colors round-robin from the palette
- If neither column is set, all objects get `COLORS[0]`

**3. Density histogram (useMemo):**
- 100 buckets across [tsMin, tsMax]
- One pass over rows to count per bucket
- Normalize to [0, 1] range
- Render as an SVG with vertical bars behind the timeline slider

**4. Visible rows filtering (useMemo depending on windowStart, windowEnd):**
- Filter rows where `Number(row[timestampCol]) >= windowStart && <= windowEnd`
- Must be finite

**5. GeoJSON construction (useMemo depending on visibleRows):**
- For each visible row, read `row[geometryCol]` — if it's an object, use it directly as the feature geometry
- Add a `color` property from the color map to each feature's properties
- Build a GeoJSON FeatureCollection

**6. Animation loop:**
- `playing` state (boolean), `speed` state (1, 2, 5, or 10)
- `timerRef = useRef<ReturnType<typeof setInterval>>(null)`
- `speedRef = useRef(speed)` — keep speed in ref to avoid effect re-trigger
- Update speedRef.current when speed changes
- `useEffect` on `playing`:
  - If playing, start `setInterval` at 100ms (10 FPS):
    - Compute step: `(tsMax - tsMin) * 0.002 * speedRef.current` (0.2% of range per tick at 1x)
    - Advance both windowStart and windowEnd by step
    - If windowEnd > tsMax, wrap to beginning (windowStart = tsMin, windowEnd = tsMin + windowSize)
  - Cleanup: clearInterval
  - If not playing, clear interval

**7. Dual-handle timeline slider:**
- Two `<input type="range">` elements, absolutely positioned on the same track
- Both use `min={tsMin}` `max={tsMax}` `step={(tsMax-tsMin)/1000}`
- Left input: `value={windowStart}`, onChange sets windowStart (clamped to <= windowEnd)
- Right input: `value={windowEnd}`, onChange sets windowEnd (clamped to >= windowStart)
- CSS: both inputs have `position: absolute`, `width: 100%`, `pointer-events: none` on the track, `pointer-events: all` on the thumb (`input[type=range]::-webkit-slider-thumb`)
- The right input sits on top (z-index) but only its thumb captures events

**8. Controls row (below timeline):**
- Play/Pause button: toggles `playing` state. Show a simple play triangle or pause bars via unicode (&#9654; / &#9646;&#9646;)
- Speed selector: row of buttons `[1x, 2x, 5x, 10x]`, highlight active
- Time display: show current window start/end as formatted timestamps. Use `new Date(windowStart * 1000).toISOString().slice(0, 19).replace('T', ' ')` if epoch seconds, or display raw numbers if format unclear

**9. Map rendering:**
- Same pattern as ResultsMap: `MapContainer` with CartoDB dark tiles, `GeoJSON` layer
- Include the Leaflet icon fix imports (same as ResultsMap)
- Key the GeoJSON layer by `key={`${windowStart}-${windowEnd}`}` (lightweight, forces re-mount on window change)
- `pointToLayer`: render `L.circleMarker` with radius 5, fillColor from feature.properties.color, weight 1, fillOpacity 0.85
- `style` callback for non-point geometries (LineString, Polygon): `{ color: feature.properties.color, weight: 2, fillOpacity: 0.3 }`
- Include a `MapBoundsController` (like ResultsMap) that fits bounds to ALL rows on mount (not just visible rows)
- No labels, no trails, no ghost effects per user decision

**10. Layout:**
- Top: Leaflet map (flex: 1, fills available space)
- Bottom: timeline container (~80px) with histogram SVG behind, dual range inputs on top, controls row below

**GeoPlaybackWidget.css:**
- `.geo-playback` — full height flex column
- `.geo-playback__map` — flex: 1, min-height: 0
- `.geo-playback__timeline` — height ~80px, position relative, padding 8px 16px

**Edge cases to handle:**
- No timestamp column found in data: show error message "Timestamp column '{name}' not found in data"
- No geometry column found: show error message
- Empty data (0 rows): show "No data to playback"
- Single timestamp value: show static view (all data visible, timeline disabled)
