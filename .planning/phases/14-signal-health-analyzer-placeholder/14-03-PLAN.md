Create the cube file following the BaseCube pattern from `alison_flights.py`.

**Class definition:**
- `cube_id = "signal_health_analyzer"`
- `name = "Signal Health Analyzer"`
- `description = "GPS anomaly detection — rule-based + Kalman filter analysis for individual flights"`
- `category = CubeCategory.ANALYSIS`

**Inputs (from CONTEXT.md, exact spec):**
- `hex_list` (LIST_OF_STRINGS, required) — hex identifiers from AlisonFlights
- `full_result` (JSON_OBJECT, accepts_full_result=True) — extracts hex_list if connected
- `target_phase` (STRING, widget_hint="select", default="any") — takeoff/cruise/landing/any
- `classify_mode` (LIST_OF_STRINGS, widget_hint="tags", default=["all"]) — Stable, Jamming, Spoofing, Dark Target, Technical Gaps

**Outputs (from CONTEXT.md, exact spec):**
- `flight_ids` (LIST_OF_STRINGS) — hex list of flights with non-normal events matching classify_mode
- `count` (NUMBER) — count of matching flights
- `events` (JSON_OBJECT) — array of all non-normal events
- `stats_summary` (JSON_OBJECT) — count of events per category

**execute() method logic:**

1. **Extract hex_list:** From `inputs["hex_list"]` or from `inputs["full_result"]["flight_ids"]` if full_result connected. Return early with empty results if no hexes.

2. **Build coverage baseline once:** Call `get_coverage_baseline()` (cached, ~7 day lookback).

3. **Determine time range:** Use `fetch_time_range_async` from kalman module if no explicit time bounds available. For each hex, auto-detect min/max ts.

4. **For each hex in hex_list:**
   a. Fetch time range for this hex (auto-detect).
   b. **target_phase filtering:** If target_phase != "any", filter the time window:
      - Define constants: `TAKEOFF_ALT_CEILING = 5000` (ft), `CRUISE_ALT_FLOOR = 10000` (ft)
      - Fetch raw positions, segment by altitude, select the relevant time windows
      - Pass filtered time bounds to detection functions
      - For simplicity in v1: just pass full time range and filter events post-hoc by their coordinates' altitude (if available in event data). Document as a tunable.
   c. **Rule-based layer:**
      - Call `detect_integrity_events_async(hex, start, end)`
      - Call `detect_transponder_shutdowns_async(hex, start, end)`
      - For each event: `score_event(event, coverage_baseline)` then `classify_event(scored_event)`
      - Add `category` field to each event dict from classify_event result
   d. **Kalman layer:**
      - Call `classify_flight_async(hex, start, end)`
      - If classification != "normal": create unified event using `kalman_event_from_result()` helper (see Pattern 5 in RESEARCH.md)
   e. Collect all events for this hex.

5. **Filter by classify_mode:**
   - Implement `CLASSIFY_MODE_MAP` exactly as in RESEARCH.md Pattern 4:
     ```python
     CLASSIFY_MODE_MAP = {
         "Stable": set(),
         "Jamming": {"gps_jamming", "probable_jamming"},
         "Spoofing": {"gps_spoofing"},
         "Dark Target": {"transponder_off"},
         "Technical Gaps": {"coverage_hole", "ambiguous"},
     }
     ```
   - If "all" in classify_mode: keep all events (no filtering)
   - Otherwise: keep events whose `category` or `classification` is in the union of mapped categories
   - Special case "Stable": if user selects only "Stable", return hexes that had ZERO non-normal events

6. **Build outputs:**
   - `flight_ids`: unique hex codes that have at least one event after filtering
   - `count`: len(flight_ids)
   - `events`: list of all filtered event dicts (already JSON-serializable from Plans 01/02)
   - `stats_summary`: Counter of `category` field across all filtered events (use `collections.Counter`)

7. **kalman_event_from_result() helper** (inside this file):
   - Converts Kalman classify_flight_async result dict to unified event schema
   - Follow Pattern 5 from RESEARCH.md exactly
   - Include both Kalman-specific fields (n_flagged, flag_pct, n_jumps, n_alt_divergence, physics_confidence) and null rule-based fields (jamming_score, spoofing_score, coverage_score, evidence)

**Error handling:**
- Wrap per-hex analysis in try/except — log errors but continue with remaining hexes
- If a hex has no positions (empty fetch), skip it with a warning log
- Log total execution time at INFO level (same pattern as AlisonFlightsCube)
