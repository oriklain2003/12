**Key transformations (sync psycopg -> async SQLAlchemy):**
- Same pattern as Plan 01: `%(key)s` -> `:key`, cursor -> `await conn.execute(text(...))`, `cur.description` -> `result.keys()`.

**Functions to port (preserve all logic verbatim):**

1. **DB functions (sync -> async):**
   - `fetch_positions(conn, hex_code, start_ts, end_ts)` -> `async def fetch_positions_async(hex_code, start_ts, end_ts) -> list[dict]`
     - Fetches ts, lat, lon, alt_baro, alt_geom, gs, tas, track, true_heading, nac_p, nic, baro_rate, geom_rate, on_ground from positions
     - Filters: source_type='adsb_icao', on_ground=false, lat IS NOT NULL
   - `fetch_time_range(conn, hex_code)` -> `async def fetch_time_range_async(hex_code) -> tuple[datetime, datetime]`
     - SELECT min(ts), max(ts) for auto-detect mode

2. **Pure computation functions (NO changes needed, keep sync):**
   - `latlon_to_enu(lat, lon, ref_lat, ref_lon)` — coordinate conversion
   - `haversine_km(lat1, lon1, lat2, lon2)` — distance calculation
   - `kalman_filter(positions)` — constant-velocity Kalman with chi-squared innovation testing
     - Uses numpy arrays for state vectors and covariance matrices
     - Uses `scipy.linalg.inv` for innovation matrix inversion
     - Returns list of dicts with `flagged` boolean per position
   - `detect_position_jumps(positions)` — consecutive reports >30 NM in <30s
   - `detect_altitude_divergence(positions)` — |alt_baro - alt_geom| > 1000ft
   - `physics_cross_validation(positions)` — multi-sensor consistency (alt, GS/TAS, track/heading, vrate)
     - Returns dict with `confidence` float and individual check results

3. **Orchestration function (sync -> async):**
   - `classify_flight(conn, hex_code, start_ts, end_ts)` -> `async def classify_flight_async(hex_code, start_ts, end_ts) -> dict`
     - Fetches positions, runs Kalman filter, detects jumps/divergence, runs physics cross-validation
     - Returns classification dict: `{"classification": "normal"|"anomalous"|"gps_spoofing", "kalman_results": [...], "jumps": [...], "alt_divergence": [...], "physics": {...}, "positions": [...], "start": datetime, "end": datetime}`
     - If start_ts/end_ts are None, auto-detect via fetch_time_range_async
     - Aggregate evidence -> classify: normal (no issues), anomalous (some flags), gps_spoofing (many flags + physics fail)

**Critical: Do NOT modify Kalman filter constants, chi-squared thresholds, jump distance thresholds (30 NM), altitude divergence threshold (1000ft), or physics cross-validation logic.** These are tuned against known test cases.

**Datetime handling:** Keep all timestamps timezone-aware. Convert datetimes to ISO strings in the classification result dict for JSON serializability.

**Import requirements:** `import numpy as np`, `from scipy.linalg import inv`, `import math`, `from datetime import datetime, timezone`, `from sqlalchemy import text`, `from app.database import engine`.
