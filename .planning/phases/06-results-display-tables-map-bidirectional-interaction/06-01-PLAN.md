**1. Create `frontend/src/utils/geoDetect.ts`:**

Export a `GeoInfo` interface with three fields:
- `latCol: string` — the detected latitude column name
- `lonCol: string` — the detected longitude column name
- `geomCol?: string` — optional column containing GeoJSON geometry objects

Export a `detectGeoColumns(rows: unknown[]): GeoInfo | null` function:
- Guard: return null if rows is empty or first row is not an object
- Get keys from first row via Object.keys()
- Match lat column using regex: `/^(lat|latitude|lat_deg|y)$/i` AND also suffix patterns `/_lat$|_latitude$/i`
- Match lon column using regex: `/^(lon|lng|long|longitude|lon_deg|x)$/i` AND also suffix patterns `/_lon$|_lng$|_longitude$/i`
- If no lat or lon column found, return null
- Validate that the matched columns in the first row contain finite numeric values (use `isFinite(Number(value))`)
- Optionally detect geometry column using: `/^(geometry|geom|geojson|shape|the_geom)$/i`
- Return `{ latCol, lonCol, geomCol }` or `{ latCol, lonCol }` if no geom column

**1. Create `frontend/src/components/Results/ResultsTable.tsx`:**

Props interface:
```typescript
interface ResultsTableProps {
  rows: unknown[];
  truncated: boolean;
  selectedRowIndex: number | null;
  onRowSelect: (index: number) => void;
}
```

Implementation:
- Auto-detect columns from `Object.keys(rows[0])` via `useMemo` — guard for empty rows returning empty array
- Sort state: `useState<string | null>(null)` for sortCol, `useState<'asc' | 'desc'>('asc')` for sortDir
- Sorted rows via `useMemo`: if no sortCol, return rows as-is. Otherwise spread+sort using `String(val ?? '').localeCompare(String(val2 ?? ''), undefined, { numeric: true })`. Flip for desc.
- `useRef<(HTMLTableRowElement | null)[]>([])` for row refs array
- `useEffect` watching `selectedRowIndex`: when non-null, call `rowRefs.current[selectedRowIndex]?.scrollIntoView({ block: 'nearest' })` — direct scroll, no animation per user decision
- **Reset sort state when data changes**: `useEffect(() => { setSortCol(null); setSortDir('asc'); }, [rows])` — prevents stale sort key from a previous cube's data (Pitfall 6 from RESEARCH.md)
- Header click handler: if clicking same column, toggle dir; if different column, set new col with 'asc'
- Sort indicator: show ` ↑` for asc, ` ↓` for desc next to active sort column header text
- Truncation warning: if `truncated` is true, render a `div.results-table__truncation-warning` with text "Showing first 100 rows" above the table scroll area
- Selected row class: `results-table__row--selected` applied when `selectedRowIndex === i`
- Each row gets onClick calling `onRowSelect(i)`
- Cell values rendered as `String((row as Record<string, unknown>)[col] ?? '')`
- Ref assignment on each tr: `ref={(el) => { rowRefs.current[i] = el; }}`
