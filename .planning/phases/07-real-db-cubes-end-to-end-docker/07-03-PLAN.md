**Create `backend/Dockerfile`:**

```dockerfile
# Stage 1: install dependencies with uv
FROM ghcr.io/astral-sh/uv:0.9 AS uv
FROM python:3.12-slim AS builder

COPY --from=uv /uv /usr/local/bin/uv
WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev --no-install-project

# Stage 2: slim runtime
FROM python:3.12-slim AS runtime
WORKDIR /app
COPY --from=builder /app/.venv /app/.venv
COPY app/ ./app/
COPY alembic/ ./alembic/
COPY alembic.ini ./

ENV PATH="/app/.venv/bin:$PATH"

EXPOSE 8000
HEALTHCHECK --interval=10s --timeout=5s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Key details:
- Pin uv to 0.9 tag (not exact patch — more resilient)
- Copy alembic files so migrations can run inside container if needed
- Use python urllib for healthcheck (curl not in slim image)
- Do NOT bake DATABASE_URL — pydantic-settings reads env vars which docker-compose provides

**Create `frontend/Dockerfile`:**

```dockerfile
# Stage 1: build with pnpm
FROM node:24-alpine AS builder
RUN corepack enable

WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

COPY . .
RUN pnpm build

# Stage 2: nginx serving SPA
FROM nginx:alpine AS runtime
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

Key details:
- corepack enable activates pnpm from packageManager field
- Lock file is confirmed as `pnpm-lock.yaml`
- `pnpm build` produces `dist/` directory
- nginx.conf is copied from frontend/ context

**Create `frontend/nginx.conf`:**

```nginx
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    # SPA fallback — all non-file routes go to index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API proxy to backend service
    location /api/ {
        proxy_pass http://backend:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # SSE support: disable buffering for streaming endpoints
        proxy_buffering off;
        proxy_cache off;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        chunked_transfer_encoding on;
    }

    # Health endpoint proxy
    location /health {
        proxy_pass http://backend:8000/health;
    }
}
```

Key details:
- proxy_buffering off is critical for SSE to work
- proxy_http_version 1.1 + empty Connection header for keep-alive
- /health proxied so docker-compose healthcheck works from frontend too

**Create `docker-compose.yml` at project root:**

```yaml
services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    environment:
      DATABASE_URL: ${DATABASE_URL}
    ports:
      - "8000:8000"
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"]
      interval: 10s
      timeout: 5s
      retries: 3
      start_period: 10s

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "3000:80"
    depends_on:
      backend:
        condition: service_healthy
```

Key details:
- No `version:` field (deprecated in modern compose)
- DATABASE_URL read from .env at project root (docker compose auto-loads .env)
- Backend healthcheck uses python urllib (no curl in slim image)
- Frontend depends on backend being healthy before starting
- Backend on port 8000 (direct access for debugging), frontend on port 3000 (user-facing)
- No explicit network — docker compose creates a default bridge network where service names resolve

Add a `.dockerignore` in both backend/ and frontend/ to speed up builds:
